Eventos

DOMContentLoaded

Los eventos en JavaScript permiten ejecutar código en respuesta dark:text-gray-200 a acciones del usuario o del sistema, como hacer clic, mover el mouse o cargar una página.

Uno de los eventos más importantes es DOMContentLoaded, se dispara cuando el HTML del documento ha sido completamente cargado y procesado, antes de que se carguen imágenes y hojas de estilo. 

//Ejemplo:

        document.addEventListener("DOMContentLoaded", () => {
          console.log("Documento cargado");
        });

//document.addEventListener("DOMContentLoaded", callback)

  1.//document → Se refiere al documento HTML.
  2.//addEventListener("DOMContentLoaded", ...) → Escucha el evento "DOMContentLoaded".
  3.//callback → Una función que se ejecutará cuando el evento ocurra.

//¿Qué hace este código?
        
  1.//Espera a que el HTML se cargue completamente en el navegador.
  2.//Luego ejecuta la función flecha (=> { console.log("Documento cargado"); }), que simplemente imprime un mensaje en la consola.

//¿Por qué usar DOMContentLoaded?
        
  1.//Asegura que el código JS solo se ejecute cuando el DOM esté listo.
  2.//Evita errores al intentar acceder a elementos que aún no existen.
  3.//Es más rápido que window.onload, ya que no espera imágenes, estilos y otros recursos.
      

//Diferencia entre DOMContentLoaded y window.onload

1.//DOMContentLoaded se ejecuta cuando el HTML está completamente cargado, pero sin esperar imágenes y estilos.

2.//window.onload	se ejecuta cuando toda la página, incluyendo imágenes y CSS, ha sido completamente cargada.

//Ejemplo con window.onload

        window.addEventListener("load", () => {
          console.log("Todos los recursos de la página han sido cargados");
        });

//Conclusión: Si solo necesitas manipular el DOM, usa DOMContentLoaded. Si necesitas esperar imágenes y estilos, usa window.onload.

Eventos del Mouse

En JavaScript, podemos detectar interacciones del usuario con el mouse a través de eventos. Estos eventos permiten ejecutar código cuando el usuario hace clic, pasa el cursor o interactúa con un elemento en la página. 

//Ejemplos:

        const nav = document.querySelector(".navegacion");

// Evento al hacer clic en el elemento

        nav.addEventListener("click", () => {
          console.log("Navegación click");
          nav.style.backgroundColor = "blue";
        });
                  
// Evento al pasar el mouse sobre el elemento

        nav.addEventListener("mouseenter", () => {
          console.log("Entrando en navegación");
          nav.style.backgroundColor = "red";
        });
                  
// Evento al quitar el mouse del elemento

        nav.addEventListener("mouseout", () => {
          console.log("Saliendo de navegación");
          nav.style.backgroundColor = "transparent";
        });

// Evento al hacer clic y soltar el mouse sobre el elemento

        nav.addEventListener("mouseup", () => {
          console.log("Navegación mouseup");
          nav.style.backgroundColor = "green";
        });
                  
// Evento al hacer doble clic

        nav.addEventListener("dblclick", () => {
          console.log("Doble clic en navegación");
          nav.style.backgroundColor = "yellow";
        });

//Los eventos del mouse permiten interactuar con los elementos de la página en tiempo real, mejorando la experiencia del usuario.

Eventos de teclado

Los eventos de teclado permiten detectar cuándo el usuario interactúa con un campo de entrada, presionando, soltando o manipulando texto dentro de él. Estos eventos son útiles para validaciones en tiempo real o para mejorar la experiencia del usuario. 

//Ejemplo:
        const busqueda = document.querySelector(".busqueda");

// Se ejecuta al presionar una tecla

        busqueda.addEventListener("keydown", () => {
          console.log("Tecla presionada");
        });
                  
// Se ejecuta cuando se suelta una tecla

        busqueda.addEventListener("keyup", () => {
          console.log("Tecla liberada");
        });
                  
// Se activa al copiar texto dentro del campo

        busqueda.addEventListener("copy", () => {
          console.log("Texto copiado");
        });
                  
// Se activa al pegar texto en el campo

        busqueda.addEventListener("paste", () => {
          console.log("Texto pegado");
        });
                  
// Se activa al cortar texto dentro del campo

        busqueda.addEventListener("cut", () => {
          console.log("Texto cortado");
        });
                  
// Se activa cuando el usuario hace clic fuera del input

        busqueda.addEventListener("blur", () => {
          console.log("Input perdió el foco");
        });
                  
// Se ejecuta cada vez que cambia el valor del input

        busqueda.addEventListener("input", (event) => {
          if (event.target.value === "") {
            console.log("Error: campo vacío");
          }
        });  

//Diferencia entre input y keydown/keyup

1.//keydown y keyup solo detectan teclas presionadas y soltadas.
2.//input es más amplio, ya que detecta cualquier cambio en el campo, incluso si el usuario pega o borra texto sin usar el teclado.

//Conclusión:

1./*Estos eventos permiten mejorar la usabilidad de los formularios, detectar acciones del usuario
en tiempo real y aplicar validaciones dinámicas.*/

Eventos de Formularios

Los eventos en formularios permiten capturar y controlar la interacción del usuario al enviar datos. Uno de los eventos más importantes es submit, que se ejecuta cuando el usuario envía un formulario, permitiendo validar o modificar su comportamiento antes de procesarlo. 

//Ejemplo:

        const formulario = document.querySelector(".formulario");

// Captura el evento submit del formulario
        formulario.addEventListener("submit", validarFormulario);
                  
        function validarFormulario(event) {
          event.preventDefault(); // Previene el envío del formulario
                  
          console.log("Procesando formulario...");
                  
// Obtener la URL de destino del formulario
        console.log("URL de acción:", event.target.action);
                  
// Obtener el valor del input con el atributo 'name="busqueda"'
        console.log("Valor ingresado:", event.target.busqueda.value);
        }

//¿Por qué usar preventDefault()?

/*Si no se usa preventDefault(), el formulario se enviará y la página se recargará, 
perdiendo la oportunidad de validar o modificar los datos antes del envío.*/

Eventos de Scroll

El evento scroll se ejecuta página hacia arriba o abajo. Es útil para efectos de animación, carga diferida de contenido o fijar elementos en la pantalla al hacer scroll. 

//Ejemplo:
        window.addEventListener("scroll", () => {
          const premiumSection = document.querySelector(".premium");
          const posicion = premiumSection.getBoundingClientRect(); // Obtiene la posición del elemento respecto a la ventana
                  
          console.log(posicion); // Muestra la posición en la consola
                  
          if (posicion.top < 714 && posicion.top > -280) {
            console.log("El elemento está visible");
            premiumSection.classList.add("visible"); // Agrega una clase cuando el elemento es visible
          } else {
            console.log("El elemento aún no es visible");
            premiumSection.classList.remove("visible"); // Elimina la clase cuando el elemento no es visible
          }
        });

//¿Cómo Funciona?

1.//Cada vez que el usuario haga scroll, la función verifica la posición de la sección .premium.
2.//Si el elemento está dentro del rango visible de la pantalla, se le agrega la clase "visible".
3.//Si el elemento está fuera del rango visible, se le quita la clase "visible".

//Aplicaciones Comunes

✅ //Mostrar elementos solo cuando sean visibles.
✅ //Activar animaciones al hacer scroll.
✅ //Cargar contenido dinámico a medida que el usuario navega.

Evento Bubbling

El evento bubbling ocurre cuando un evento se propaga desde el elemento más específico (hijo) hacia los elementos padres, como si fuera una burbuja ascendiendo en el DOM. Esto puede causar que múltiples eventos se ejecuten al mismo tiempo de manera no deseada. 

//Ejemplos:

        const cardDiv = document.querySelector(".card");
        const infoDiv = document.querySelector(".info");
        const tituloDiv = document.querySelector(".titulo");
                  
        cardDiv.addEventListener("click", (event) => {
          event.stopPropagation(); // Detiene la propagación del evento
          console.log("Click en card");
        });
                  
        infoDiv.addEventListener("click", (event) => {
          event.stopPropagation();
          console.log("Click en info");
        });
                  
        tituloDiv.addEventListener("click", (event) => {
          event.stopPropagation();
          console.log("Click en titulo");
        });

//Explicación:

/*Si no usamos event.stopPropagation(), al hacer clic en el elemento más interno .titulo, 
el evento también se propagaría hacia sus elementos padres .info y luego .card, 
generando múltiples registros en la consola.*/

//Casos de usamos

1.//Evitar que múltiples eventos se ejecuten cuando solo queremos capturar el clic en un elemento específico.
2.//Controlar mejor la ejecución de eventos en estructuras anidadas del DOM.

Delegación de Eventos

La delegación de eventos es una técnica en la que en lugar de asignar eventos a múltiples elementos hijos individualmente, asignamos el evento a un elemento padre común y detectamos en qué hijo se hizo clic usando event.target. 

//Ejemplos:

        const cardDiv2 = document.querySelector(".card");

        cardDiv2.addEventListener("click", (event) => {
          if (event.target.classList.contains("titulo")) {
            console.log("Diste click en título");
          } else if (event.target.classList.contains("info")) {
            console.log("Diste click en info");
          } else if (event.target.classList.contains("card")) {
            console.log("Diste click en card");
          }
        });

//Explicación:

1.//Asignamos el evento click al elemento padre (.card).
2.//Usamos event.target para saber en qué hijo se hizo clic.
3.//Detectamos la clase del elemento clickeado con .classList.contains().

//¿Por Qué Usar Delegación de Eventos?

🔹 //Optimiza el rendimiento en listas largas o elementos dinámicos.
🔹 //Evita asignar eventos repetidos a múltiples elementos.
🔹 //Facilita el mantenimiento del código.

Evitar la propagación con contenido creado

Cuando creamos elementos dinámicamente en JavaScript y les asignamos eventos, estos pueden propagarse de manera no deseada si no controlamos el flujo de los eventos. 

//Ejemplo:

// Crear un párrafo con clase "categoria"

        const parrafo1 = document.createElement("P");
        parrafo1.textContent = "Concierto";
        parrafo1.classList.add("categoria", "concierto");
                  
// Segundo párrafo con clase "titulo"

        const parrafo2 = document.createElement("P");
        parrafo2.textContent = "Concierto de Rock";
        parrafo2.classList.add("titulo");
                  
// Tercer párrafo con clase "precio" y evento de click

        const parrafo3 = document.createElement("p");
        parrafo3.textContent = "$800 por persona";
        parrafo3.classList.add("precio");
        parrafo3.onclick = (event) => {
          event.stopPropagation(); // Evita que el evento se propague al contenedor padre
          nuevaFuncion(1);
        };
                  
// Crear el contenedor de información y agregar los párrafos

        const info = document.createElement("div");
        info.classList.add("info");
        info.append(parrafo1, parrafo2, parrafo3);
                  
// Crear la imagen

        const imagen = document.createElement("img");
        imagen.src = "img/hacer2.jpg";
                  
// Crear el card y agregar la imagen e información

        const contenedorCard = document.createElement("div");
        contenedorCard.classList.add("contenedorCard");
        contenedorCard.append(imagen, info);
                  
// Insertar el card en el HTML

        const contenedor = document.querySelector(".hacer .contenedor-cards");
        contenedor.appendChild(contenedorCard);
                  
// Función que se ejecuta al hacer clic en el precio

        function nuevaFuncion(id) {
          console.log("Funcion nueva", id);
        }

//¿Cómo Funciona?

1.//Creamos elementos dinámicamente, como párrafos (p), una imagen (img) y un contenedor (div).
2.//Agregamos eventos, como el onclick en el párrafo de precio.
3.//Usamos event.stopPropagation() dentro de parrafo3.onclick para evitar que el evento se propague al contenedor padre.
4.//Añadimos los elementos al DOM usando appendChild() y append().

//¿Por Qué Usar stopPropagation()?

🔹 //Evita que el evento se propague a elementos padres.
🔹 //Permite un control más preciso sobre qué elementos activan eventos.
🔹 //Mejora la usabilidad y previene comportamientos inesperados.
