Funciones

Declaración y uso de Funciones.

Las funciones son bloques de código reutilizables que pueden ejecutarse en cualquier momento al ser llamadas. Son una herramienta fundamental en JavaScript, ya que permiten estructurar y modularizar el código. 

//Ejemplo:
        function sumar() {
          console.log(2 + 2);
        }
                  
        sumar(); // Resultado: 4

//Expresiones de funciones (function expressions)

/*Una expresión de función consiste en asignar una función anónima (sin nombre) a una variable o constante.
Luego, podemos invocar la función a través de esa variable o constante.*/

        const sumar2 = function () {
          console.log(3 + 3);
        };

        sumar2(); // Resultado: 6

/*Las funciones son esenciales para reutilizar y organizar el código. Utiliza declaraciones de función
para bloques de código generales y expresiones de función cuando necesites más flexibilidad, como asignarlas
a variables o utilizarlas en contexto dinámico.*/

Llamar una función antes de declararla

En JavaScript, el hoisting (elevación) es un comportamiento del lenguaje en el cual las declaraciones de variables y funciones se mueven a la parte superior de su contexto durante la fase de compilación, antes de que el código se ejecute. Esto afecta cómo y cuándo puedes llamar a las funciones. 

/*Las funciones declaradas con el formato function nombreFunción() { ... } son completamente hoisted, lo que significa 
que puedes llamarlas antes de declararlas en el código.*/

        sumar(); // Esto funciona debido al hoisting.

        function sumar() {
          console.log(2 + 2);
        }

/*Explicación:

Durante la fase de compilación, el motor de JavaScript eleva la declaración de la función sumar al inicio del archivo
o del contexto en el que fue definida. Por lo tanto, puedes llamar a la función incluso antes de que aparezca en el código.*/

//Expresiones de funciones (function expressions)

/*Las expresiones de funciones asignadas a variables, como const o let, no son hoisted de la misma manera. 
Aunque el identificador de la variable es elevado, no se asigna el valor de la función hasta que la línea
de asignación se ejecuta. Por lo tanto, intentar llamar a la función antes de su definición genera un error.*/

        sumar2(); // Error: Cannot access 'sumar2' before initialization.

        const sumar2 = function () {
          console.log(3 + 3);
        };

/*Conclusión

- Usa declaraciones de funciones cuando necesites llamarlas en cualquier lugar del código, incluso antes de que aparezcan.

- Usa expresiones de funciones cuando desees un comportamiento más controlado y asignarlas a variables o usarlas de forma dinámica.*/

Diferencia entre metodo y función

En JavaScript, funciones y métodos son bloques de código reutilizables, pero tienen diferencias clave en cómo y dónde se usan, así como en el contexto en el que se invocan. 

//La diferencia entre funciones y metodos es principalmente el contexto en el que son utilizadas

const numero1 = 20;
const numero2 = "20";

console.log(numero1.toString()); //Metodo
console.log(parseInt(numero2)); //Funcion

//El metodo se lo llama de manera directa, pero la funcion se lo llama de manera indirecta

function sumar() {
  console.log(2 + 2);
}

sumar();

/*Conclusión:

Usa funciones cuando el bloque de código no dependa de un objeto específico.

Usa métodos cuando el bloque de código sea específico a un objeto o se aproveche del contexto (this) del objeto al que pertenece.*/

Parametros y argumentos de las funciones

Al trabajar con funciones en JavaScript, es importante entender la diferencia entre parámetros y argumentos, ya que son conceptos clave para el funcionamiento de las funciones. 

//Parametros:

/*Los parámetros son los nombres de las variables que se definen entre los paréntesis al declarar una función.
Actúan como "contenedores" que recibirán los valores que se pasen a la función al llamarla.*/

        function sumar(a, b) {
          // `a` y `b` son parámetros
          console.log(a + b);
        }

// En este caso, a y b son los parámetros de la función sumar.
// No tienen un valor fijo; su valor dependerá de los argumentos que se les pasen al llamar la función.

//Argumentos:

/*Los argumentos son los valores reales que se pasan a una función cuando esta es invocada.
Estos valores son asignados a los parámetros definidos en la función.*/

        sumar(2, 3); // `2` y `3` son argumentos

//Codigo completo y explicación:

        function sumar(a, b) {
          // `a` y `b` son parámetros
          console.log(a + b);
        }

        sumar(2, 3); // `2` y `3` son argumentos
        sumar(500, 150); // Aquí los argumentos son `500` y `150`

//Ejemplo 2:

        function saludar(nombre, apellido) {
          // `nombre` y `apellido` son parámetros
          console.log(`Hola ${nombre} ${apellido}`);
        }

        saludar("Sebastián", "Salinas"); // Los argumentos son "Sebastián" y "Salinas"

/*Resumen:

- Parámetros: Variables definidas en la declaración de la función; actúan como "marcadores de posición".

- Argumentos: Valores reales que se pasan a la función cuando es invocada.

Esto permite que las funciones sean reutilizables y dinámicas, ya que los valores de entrada (argumentos) pueden variar según el caso.*/

Comunicación entre funciones

En JavaScript, las funciones pueden interactuar y comunicarse entre sí. Esto se logra llamando a una función desde otra función. Este enfoque es fundamental para estructurar código modular, reutilizable y más fácil de mantener. 

//Ventajas de dividir el código en funciones:

1.// Modularidad: El código se divide en bloques pequeños y manejables.
2.// Reutilización: Las funciones pueden ser llamadas en múltiples lugares sin duplicar el código.
3.// Legibilidad: Es más fácil entender el flujo de un programa cuando se organiza en funciones.

        iniciarApp();

        function iniciarApp() {
          console.log("Iniciando App");

          segundaFuncion(); // Llamada a la segunda función
        }

        function segundaFuncion() {
          console.log("Desde la segunda función");

          usuarioAutenticado("Sebastián Salinas"); // Llamada a la tercera función con un argumento
        }

        function usuarioAutenticado(usuario) {
          console.log("Autenticando usuario...espere");
          console.log(`Usuario autenticado exitosamente: ${usuario}`);
        }

//Explicación paso a paso:

1.// Llamada inicial:

        iniciarApp();

2.// Función iniciarApp:

  // - Muestra el mensaje "Iniciando App".
  // - Llama a la función segundaFuncion desde dentro de su bloque.

3.// Función segundaFuncion:

  // - Muestra el mensaje "Desde la segunda función".
  // - Llama a la función usuarioAutenticado, pasándole un argumento: "Sebastián Salinas".

4.// Función usuarioAutenticado:

  // - Recibe el argumento "Sebastián Salinas" y lo asigna al parámetro usuario.
  // - Imprime los mensajes "Autenticando usuario...espere" y "Usuario autenticado exitosamente: Sebastián Salinas".

//Resultado en consola:

        /*Iniciando App
        Desde la segunda función
        Autenticando usuario...espere
        Usuario autenticado exitosamente: Sebastián Salinas*/

Funciones que retornan valores

Las funciones pueden devolver un resultado utilizando la palabra clave return. Esto permite almacenar el valor devuelto en una variable o utilizarlo directamente en otras operaciones. Este enfoque es fundamental para crear programas más dinámicos y reutilizables. 

//Ejemplo:
        function sumar(a, b) {
          return a + b; // Devuelve la suma de los parámetros a y b
        }
        
// Guardamos el valor retornado en una variable

        const resultado = sumar(2, 3);
  
// Mostramos el resultado 

        console.log(resultado); // 5

/*Explicación:

1. La función sumar recibe dos parámetros (a y b).
2. Utiliza la palabra clave return para devolver la suma de a y b.
3. Al llamar la función, el valor retornado (5) se asigna a la variable resultado.
4. Imprimimos el contenido de resultado en la consola.*/

//Ejemplo carrito de compras:

        let total = 0;

        function agregarCarrito(precio) {
          return (total += precio); // Suma el precio al total y lo devuelve
        }

        function añadirImpuesto(total) {
          return total * 1.15; // Calcula el total con un impuesto del 15%
        }

// Agregamos productos al carrito
        total = agregarCarrito(100);
        total = agregarCarrito(200);
        total = agregarCarrito(300);

// Calculamos el total con impuestos
        const totalConImpuestos = añadirImpuesto(total);

// Mostramos el total con y sin impuestos
        console.log(`El total a pagar es de ${totalConImpuestos}€`); // 690€

/*Explicación:

1. Función agregarCarrito:
    Suma el precio al valor acumulado de total y devuelve el nuevo total.

2. Función añadirImpuesto:
    Multiplica el total por 1.15 para calcular un 15% de impuesto y devuelve el resultado.

3. Proceso:
    Se agregan tres productos al carrito con precios de 100, 200 y 300.
    Se calcula el total con impuestos, asignándolo a totalConImpuestos.
    Se imprime el total final en la consola.*/

/*Conclusión:

Las funciones que retornan valores permiten encadenar cálculos y reutilizar lógica. Este enfoque es ideal para manejar operaciones
como cálculos matemáticos, acumulaciones de datos y generación dinámica de valores. Dividir el código en funciones específicas y 
bien definidas mejora la claridad y la flexibilidad del programa.*/

Funciones dentro de un objeto

En JavaScript, un objeto puede contener métodos (funciones asociadas a ese objeto) que permiten realizar acciones relacionadas con sus propiedades o su contexto. Esto es útil para organizar el código y agrupar funcionalidades relacionadas bajo un mismo objeto. 

//Ejemplo Reproductor de música:

        const reproductor = {
          reproducir: function (id) {
            console.log(`Reproduciendo canción con el ID ${id}`);
          },
          pausar: function () {
            console.log("Pausando");
          },
          borrar: function (id) {
            console.log(`Borrando canción con el ID ${id}`);
          },
          crearPlaylist: function (nombre) {
            console.log(`Creando una playlist del género ${nombre}`);
          },
          reproducirPlaylist: function (nombre) {
            console.log(`Reproduciendo la playlist de ${nombre}`);
          },
        };
        
// Llamadas a los métodos

        reproductor.reproducir(30);
        reproductor.reproducir(10);
        reproductor.pausar();
        reproductor.borrar(30);
        reproductor.crearPlaylist("Rock nacional");
        reproductor.crearPlaylist("Cumbia");
        reproductor.reproducirPlaylist("Rock nacional");

//Explicación del código:

1. //Objeto reproductor:

      /*Representa un reproductor de música que puede realizar acciones como reproducir, pausar, borrar canciones, y gestionar playlists.

      Cada acción se define como un método del objeto.*/

2. /*Métodos del objeto:

      - reproducir(id)
          Recibe un id como argumento e imprime un mensaje indicando que se está reproduciendo la canción con ese ID.

      - pausar()
          Imprime "Pausando", simulando la pausa de una canción.

      - borrar(id)
          Recibe un id como argumento y muestra un mensaje indicando que la canción con ese ID se ha borrado.

      - crearPlaylist(nombre)
          Recibe el nombre de un género y crea una playlist asociada, mostrando un mensaje con el género.

      - reproducirPlaylist(nombre)
          Recibe el nombre de una playlist y muestra un mensaje indicando que se está reproduciendo esa playlist.*/

3. /*Llamadas a los métodos:

      - reproducir(30) y reproducir(10)
          Llaman al método reproducir con los IDs 30 y 10, mostrando mensajes específicos para cada canción.

      - pausar()
          Llama al método pausar, que imprime "Pausando".

      - borrar(30)
          Llama al método borrar con el ID 30, indicando que la canción con ese ID se eliminó.

      - crearPlaylist("Rock nacional") y crearPlaylist("Cumbia")
          Crea dos playlists con los nombres "Rock nacional" y "Cumbia".

      - reproducirPlaylist("Rock nacional")
          Reproduce la playlist "Rock nacional".*/

/*Conclusión:

Definir funciones dentro de un objeto es una práctica poderosa para estructurar y encapsular lógica relacionada.
Esto mejora la legibilidad, facilita el mantenimiento y permite una mayor modularidad del código.*/

Arrow Functions

Las arrow functions son una forma más moderna y concisa de declarar funciones en JavaScript, introducida con ES6. Su sintaxis simplificada las hace ideales para funciones de una o pocas líneas, especialmente cuando trabajamos con funciones de callback o funciones anónimas. 

/*Ventajas de las Arrow Functions:

1. Sintaxis más corta: No necesitan la palabra clave function.

2. Implicación del retorno: Si el cuerpo de la función es una sola línea, el valor es retornado implícitamente (sin usar return).
        
3. Contexto de this: Mantienen el valor de this del contexto donde fueron declaradas, lo que evita problemas comunes
al usar funciones normales en objetos o callbacks.*/

//Ejemplo:

// Declarar una función con Arrow Function

        const sumar = (a, b) => a + b; // Sintaxis corta para retornar a + b
        console.log(sumar(3, 5)); // Imprime 8

        /*Parámetros: (a, b) son los valores que se pasan a la función.
          Cuerpo: La expresión a + b es evaluada y el resultado se retorna implícitamente.
          Ventaja: Sin necesidad de escribir function o return.*/

// Declarar una función normal

        const aprendiendo = function () {
          console.log("Aprendiendo JavaScript");
        };

// Llamar a la función aprendiendo

        aprendiendo();

// Declarar una Arrow Function con un solo parámetro

        const aprendiendo2 = (tecnologia) => `Aprendiendo ${tecnologia}`;

          /*Un solo parámetro: Cuando hay un solo parámetro, los paréntesis son opcionales.
            Retorno implícito: Como la función es de una sola línea, no se necesitan llaves {} ni la palabra clave return.*/

// Llamar a la función aprendiendo2

        console.log(aprendiendo2("JavaScript")); // Imprime: Aprendiendo JavaScript

//Resumen de sintaxis de Arrow Functions:

1. //Con múltiples parámetros y un cuerpo corto:

        const sumar = (a, b) => a + b;

2. //Con un solo parámetro:

        const cuadrado = x => x * x;

3. /*Con un cuerpo más complejo: Cuando el cuerpo de la función tiene varias líneas, usamos llaves {} 
y debemos incluir explícitamente return para devolver un valor.*/

        const calcularArea = (base, altura) => {
          const area = (base * altura) / 2;
          return area;
        };

/*Ejemplos de arrow functions con .forEach() y .map():*/

        const carrito = [
          { nombre: "Monitor Sammsung 24 pulgadas", precio: 110 },
          { nombre: "Notebook Acer Nitro 5", precio: 1100 },
          { nombre: "Teclado Logitech", precio: 15 },
          { nombre: "Mouse Logitech", precio: 80 },
          { nombre: "Poco X3 Pro", precio: 250 },
          { nombre: "Play Station 4", precio: 150 },
        ];

        carrito.forEach((producto) =>
        console.log(`${producto.nombre} - Precio: ${producto.precio}`)
        );

        const resultados = carrito.map((producto) =>
          console.log(`${producto.nombre} - Precio: ${producto.precio}`)
        );

/*Cuándo usar Arrow Functions:

- Funciones cortas: Ideales para callbacks, funciones simples y operaciones pequeñas.

- Retorno implícito: Cuando necesitas que una función retorne rápidamente un valor calculado.

- Evitar problemas de this: En métodos que necesitan preservar el contexto de this del objeto padre.

Sin embargo, para métodos en objetos o en clases que requieren manipular this, es mejor usar
funciones normales, ya que las arrow functions no tienen su propio this.*/

Objetos con funciones y métodos avanzados

El siguiente ejemplo define un objeto llamado reproductor, que incluye métodos, propiedades, y utiliza características avanzadas como set y get. 

//Ejemplo:

        const reproductor = {
          cancion: "", // Propiedad inicial
        
// Métodos del objeto

          reproducir: (id) => console.log(`Reproduciendo canción con el id ${id}`),
          pausar: () => console.log("Pausando"),
          borrar: (id) => console.log(`Borrando canción con el id ${id}`),
          crearPlaylist: (nombre) =>
            console.log(`Creando una playlist del género ${nombre}`),
          reproducirPlaylist: (nombre) =>
            console.log(`Reproduciendo la playlist de ${nombre}`),
        
// Setter: define o actualiza una propiedad del objeto

          set nuevaCancion(cancion) {
            this.cancion = cancion; // `this` hace referencia al objeto actual
            console.log(`Añadiendo canción: ${cancion}`);
          },
        
// Getter: accede al valor de una propiedad del objeto

          get obtenerCancion() {
            console.log(`${this.cancion} ya se encuentra añadida al reproductor`);
          },
        };
        
// Uso del setter para agregar una nueva canción

        reproductor.nuevaCancion = "Seminare";
        
// Uso del getter para acceder a la canción actual

        reproductor.obtenerCancion;
        
// Llamada a métodos del objeto

        reproductor.reproducir(30);
        reproductor.reproducir(10);
        reproductor.pausar();
        reproductor.borrar(30);
        reproductor.crearPlaylist("Rock nacional");
        reproductor.crearPlaylist("Cumbia");
        reproductor.reproducirPlaylist("Rock nacional");

//Puntos clave:

  1. /*Propiedad inicial:
        cancion es una propiedad que se puede modificar usando un setter y consultar usando un getter.*/

  2. /*Métodos del objeto:
        Son funciones asociadas al objeto reproductor, como reproducir o crearPlaylist, que ejecutan acciones
        relacionadas con el reproductor.*/

  3. /*Uso de set:
        Permite asignar un valor a la propiedad cancion.
        Cuando usamos reproductor.nuevaCancion = "Seminare", internamente se ejecuta el código dentro de set nuevaCancion.*/

  4. /*Uso de get:
        Proporciona una forma de acceder al valor de la propiedad cancion de manera controlada.
        Al llamar a reproductor.obtenerCancion, ejecutamos el código dentro de get.*/

  5. /*Uso de this:
        Dentro de los métodos, this se refiere al objeto actual (reproductor), asegurando que las operaciones
        se realicen sobre sus propiedades y métodos.*/

  6. /*Arrow functions:
        Todos los métodos están definidos con sintaxis de arrow function, que es más concisa, pero set y get 
        usan la función tradicional porque necesitan el contexto de this.*/

/*Resumen:

- set: Útil para controlar cómo se actualizan las propiedades de un objeto.

- get: Ideal para personalizar cómo se accede a los valores de las propiedades.

- this: Referencia al objeto actual.

- Métodos: Realizan acciones específicas relacionadas con el objeto.

- Arrow functions: Simplifican la sintaxis de los métodos.*/
