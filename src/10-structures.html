Estructuras de control

Declaración y uso de Estructuras de Control

La estructura de control if solo si se cumple una condición especificada. Si la condición no es verdadera, se puede definir un bloque de código alternativo con else. Esta es una forma básica de toma de decisiones en programación. 

//Ejemplo:

        const puntaje = 1000;  // Definimos una variable "puntaje" con valor 1000

        if (puntaje < 1000) {  // Si "puntaje" es menor que 1000
          console.log("Tu no llega a el minimo requerido");  // Este bloque se ejecuta si la condición es verdadera
        } else {  // Si "puntaje" NO es menor que 1000 (es decir, si es 1000 o más)
          console.log("¡Felicidades! Has superado el puntaje");  // Este bloque se ejecuta si la condición es falsa
        }

/*¿Cuándo usar if?

Usamos if cuando necesitamos tomar una decisión y ejecutar un bloque de código dependiendo de si una condición es verdadera o falsa.
        
if: Ejecuta el bloque de código si la condición es verdadera.

else: Ejecuta el bloque de código si la condición es falsa.

Este tipo de estructura es fundamental cuando necesitas realizar diferentes acciones en función de ciertos criterios o valores.*/

Estructura de control else if

La estructura de control else if permite realizar múltiples verificaciones de condiciones en secuencia. Es útil cuando necesitamos comprobar varias condiciones y ejecutar diferentes bloques de código en función de cuál de ellas sea verdadera. 

//Ejemplo:

          const efectivo = 100;  // Definimos la cantidad de dinero en efectivo
          const totalAPagar = 300;  // Definimos el monto total que debe pagarse
          const tarjeta = true;  // Definimos si la persona tiene una tarjeta de débito
          const tarjetaCredito = true;  // Definimos si la persona tiene una tarjeta de crédito
                  
          if (efectivo >= totalAPagar) {  // Verificamos si el efectivo es suficiente
            console.log("Podemos realizar el pago");
          } else if (tarjeta || tarjetaCredito) {  // Si no hay suficiente efectivo, verificamos si hay tarjeta de débito o crédito
            console.log("Podemos realizar el pago");
          } else {  // Si no hay efectivo y no hay tarjetas disponibles
            console.log("No podemos realizar el pago");
          }

/*
¿Cuándo usar else if?

Usamos else if cuando tenemos varias condiciones mutuamente excluyentes y necesitamos ejecutar diferentes 
bloques de código en función de cuál de esas condiciones se cumpla.

if: Verifica la primera condición.

else if: Si la primera condición no se cumple, se evalúa esta segunda condición (y puede haber más).

else: Si ninguna de las condiciones anteriores es verdadera, se ejecuta el bloque de código dentro del else.

El uso de else if es útil cuando tenemos varias condiciones alternativas y solo necesitamos ejecutar un 
bloque de código según la primera condición verdadera.*/

Estructura de control switch

El switch es una estructura de control que nos permite realizar una evaluación sobre una variable o expresión y ejecutar diferentes bloques de código dependiendo de su valor. Es especialmente útil cuando tenemos múltiples opciones y queremos manejar diferentes casos de forma más clara y ordenada, evitando anidaciones complejas de if y else if. 

//Ejemplo:

        const tipoDeUsuario = "admin";  // Definimos el tipo de usuario

        switch (tipoDeUsuario) {  // Evaluamos la variable tipoDeUsuario
          case "admin":  // Si tipoDeUsuario es "admin"
            console.log("¡Bienvenido admin");
            break;  // Salimos del switch
                    case "usuario":  // Si tipoDeUsuario es "usuario"
            console.log("Bienvenido usuario");
            break;  // Salimos del switch
          default:  // Si tipoDeUsuario no es ni "admin" ni "usuario"
            console.log("Bienvenido sin usuario");
        }

/*Nota:

El break se utiliza para salir de la estructura switch una vez que se haya ejecutado el 
bloque de código correspondiente. Si omites el break, el flujo de ejecución continuará y 
ejecutará todos los casos siguientes (lo que se conoce como "fall-through").*/

/*Cuándo usar switch?

El switch es ideal cuando tenemos una variable o expresión con múltiples posibles valores y
queremos manejar cada valor con una acción diferente. Es más eficiente y legible que usar 
múltiples if y else if, especialmente cuando tenemos muchas opciones.*/

Operador ternario

El operador ternario es una forma concisa de realizar operaciones condicionales en JavaScript.Es útil para condiciones simples, donde se evalúa si una expresión es verdadera o falsa, y se ejecuta una de dos opciones. 

//Sintaxis:

        condición ? expresión_si_verdadero : expresión_si_falso;

//Ejemplo:

        const autenticado = true;
        const puedePagar = true;
                  
        autenticado && puedePagar ? console.log("El usuario está autenticado"): console.log("El usuario no está autenticado");

/*Cuándo usar el operador ternario

- Condiciones simples que no requieren lógica compleja.
- Mantener el código más conciso y limpio.

No es recomendable usarlo cuando:

- Las condiciones son complejas o difíciles de leer.
- Hay múltiples condiciones anidadas, ya que puede afectar la claridad del código.*/

/*Conclusión: Aunque el operador ternario puede ser práctico para expresiones simples, no es la mejor opción
para condiciones complejas. En esos casos, es preferible utilizar estructuras como if-else para mantener
el código más legible y fácil de mantener.*/

Buenas prácticas al trabajar con estructuras de control

Al trabajar con estructuras de control como if, else if, else y switch, es importante seguir buenas prácticas para mejorar la legibilidad y mantenibilidad del código. Aquí te dejo algunas recomendaciones y un ejemplo de cómo usarlas: 

1. //Evitar anidar múltiples niveles de if innecesarios:

/*En lugar de tener varias declaraciones if anidadas, es preferible usar un return temprano dentro de una función
cuando se cumple una condición. Esto hace que el código sea más limpio y evita una lectura compleja.*/

2. //Utilizar return para salir de la función temprano:

/*En las funciones, usar return cuando ya no es necesario seguir ejecutando el código ayuda a evitar
procesamiento innecesario. Esto mejora la eficiencia y mantiene el código organizado.*/

        const autenticado = true;

        if (autenticado) {  // Verifica si la variable 'autenticado' es verdadera
          console.log("autenticado");  // Si es true, muestra "autenticado"
        }

        const puntaje = 500;

        function revisarPuntaje() {
          if (puntaje > 400) {  // Verifica si el puntaje es mayor que 400
            console.log("¡Excelente puntaje!");  // Si es mayor, muestra el mensaje
            return;  // Sale de la función para evitar más evaluaciones innecesarias
          } else if (puntaje > 300) {  // Si no, verifica si el puntaje es mayor que 300
            console.log("Buen puntaje");  // Si es mayor, muestra el mensaje
            return;  // Sale de la función
          }
        }

// Al llamar la función, esta evalúa el puntaje y muestra el mensaje correspondiente

        revisarPuntaje();  // Imprimirá "¡Excelente puntaje!" porque el puntaje es 500

/*¿Por qué es una buena práctica usar return temprano?

- Evita código innecesario: Si ya se cumplió una condición, no es necesario continuar evaluando las demás.

- Hace el código más legible: El flujo de ejecución es más claro, ya que no hay múltiples anidamientos de if.

- Mejora la eficiencia: El código deja de ejecutarse una vez que se cumple la condición, lo que puede 
ser beneficioso si las funciones contienen operaciones costosas.*/

3.// Usar condiciones claras y concisas:

// Mala práctica (condición complicada)

        if (a > 10 && b < 20 || c === 5) {
          console.log('Condición compleja');
        }

// Buena práctica (condición clara)

        const condicionesCumplidas = a > 10 && b < 20 || c === 5;
        if (condicionesCumplidas) {
          console.log('Condición cumplida');
        }

4.// Evitar el uso excesivo de else:

// Mala práctica (uso innecesario de else)

        if (edad > 18) {
          console.log('Mayor de edad');
        } else {
          console.log('Menor de edad');
        }

// Buena práctica (sin uso de else)

        const resultado = edad > 18 ? 'Mayor de edad' : 'Menor de edad';
        console.log(resultado);

5.// Evitar los "nested if" innecesarios (anidar if demasiado profundo):

// Mala práctica (anidamiento innecesario)

        if (usuarioActivo) {
          if (usuarioLogueado) {
            if (usuarioVerificado) {
              console.log("Acceso permitido");
            }
          }
        }

// Buena práctica (simplificar la lógica)

        if (usuarioActivo && usuarioLogueado && usuarioVerificado) {
          console.log("Acceso permitido");
        }

6.// Usar operadores ternarios para condiciones simples:

/*El operador ternario condición ? valor_si_true : valor_si_false es útil cuando tienes una simple condición con dos opciones.*/

// Mala práctica (if-else innecesario)

        if (edad >= 18) {
          mensaje = 'Mayor de edad';
        } else {
          mensaje = 'Menor de edad';
        }

// Buena práctica (uso del operador ternario)

        const mensaje = edad >= 18 ? 'Mayor de edad' : 'Menor de edad';

7.// Utilizar switch cuando tienes múltiples condiciones que verifican el mismo valor:

/*Si tienes múltiples if-else que verifican diferentes valores de una misma variable, 
el uso de switch mejora la legibilidad y el rendimiento.*/

// Mala práctica (múltiples if-else)

        if (rol === 'admin') {
          console.log('Acceso completo');
        } else if (rol === 'editor') {
          console.log('Acceso parcial');
        } else if (rol === 'viewer') {
          console.log('Acceso solo lectura');
        }

// Buena práctica (usando switch)

        switch (rol) {
          case 'admin':
            console.log('Acceso completo');
            break;
          case 'editor':
            console.log('Acceso parcial');
            break;
          case 'viewer':
            console.log('Acceso solo lectura');
            break;
          default:
            console.log('Rol desconocido');
        }

8. //No usar == o !=, usar siempre === o !==

/*El uso de == y != realiza una conversión de tipo implícita, lo que puede producir resultados inesperados.
Usa siempre === y !== para comparar valores y tipos exactos.*/

// Mala práctica (usando ==)

        if (a == '10') {
          console.log('A es igual a 10');
        }

// Buena práctica (usando ===)

        if (a === 10) {
          console.log('A es igual a 10');
        }

9. //Evitar condiciones complejas con muchos operadores lógicos

/*Si tienes una condición compleja con múltiples operadores && o ||, es recomendable dividirla
en varias condiciones más sencillas o usar funciones auxiliares para mejorar la claridad.*/

// Mala práctica (condición compleja)

        if (a > 10 && b < 5 || c === 10 && d !== 3) {
          console.log('Condición compleja');
        }

// Buena práctica (dividir en condiciones más sencillas)

        const condicion1 = a > 10 && b < 5;
        const condicion2 = c === 10 && d !== 3;

        if (condicion1 || condicion2) {
          console.log('Condición cumplida');
        }

10. // Utilizar objetos para condiciones múltiples

/*Cuando tienes que evaluar varias condiciones relacionadas con la misma entidad, usar un objeto para
almacenar esos valores puede ser más eficiente y organizado.*/

/*Seguir estas buenas prácticas ayuda a hacer el código más limpio, legible y fácil de mantener.
Además, contribuye a mejorar la eficiencia, evitando el uso innecesario de recursos y facilitando 
la depuración del código. Siempre recuerda que escribir código claro y conciso es clave para 
proyectos colaborativos y escalables.*/
